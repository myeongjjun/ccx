# Launcher Flow

## Workflow

1. start from the PRD interaction loop: open, type, navigate, enter, focus
2. keep the result list compact and information-dense
3. make focus transfer a first-class success criterion
4. defer non-MVP ideas like previews and favorites

## Tool Usage

- describe UI states separately from platform integration details
- validate that every row field is backed by indexed session data

## Output Rules

- proposals must preserve the product scope as an iTerm2 companion
- keyboard shortcuts and row layout should be explicit
